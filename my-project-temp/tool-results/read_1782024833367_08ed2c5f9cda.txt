   387Ôćĺ## Stage Summary
   388Ôćĺ
   389Ôćĺ**Verdict: NOT launch-ready.** Apical is an impressive demo with a complete-looking feature surface, but it has fundamental security and configuration problems that block production launch.
   390Ôćĺ
   391Ôćĺ**What's solid:**
   392Ôćĺ- The platform layer (`src/lib/platform/`) is well-architected: 9 services with clear boundaries, proper encryption (for BYOK + DataConnections), a real ReAct agent loop, a zero-dep SMTP client, Stripe webhook signature verification, and a model registry spanning 13 providers.
   393Ôćĺ- The 4 mini-services are properly packaged and runnable.
   394Ôćĺ- The Prisma schema (36 models) is comprehensive and well-commented.
   395Ôćĺ- The landing page is polished.
   396Ôćĺ- The agent chat + workflow proposal + research + script analysis flow is genuinely functional.
   397Ôćĺ- The dev-bypass mode lets the app work end-to-end without any external setup.
   398Ôćĺ
   399Ôćĺ**What's broken:**
   400Ôćĺ- **Auth is essentially absent** outside of dev-bypass mode. Most routes don't even call `withUser` ÔÇö they operate on global data or use `userId='system'` placeholders. This is the single biggest blocker.
   401Ôćĺ- **TypeScript errors are silently ignored** at build time (`ignoreBuildErrors: true`).
   402Ôćĺ- **Two credentials stores are unencrypted** (`IntegrationSession` base64, `Credential.oauth*` plaintext) despite a working `vault.ts` sitting right there.
   403Ôćĺ- **Stripe is in demo mode** ÔÇö paid plans upgrade instantly without payment.
   404Ôćĺ- **No actual pages exist** beyond the landing page ÔÇö `/login`, `/signup`, `/docs`, `/developer` all 404.
